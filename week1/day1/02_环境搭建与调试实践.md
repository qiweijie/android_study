## 一、AOSP 环境搭建（了解流程）

> ⚠️ **说明**：完整 AOSP 编译需要 200GB+ 磁盘空间和数小时下载时间。本节先了解流程，实际编译可根据需要后续进行。

### 1.1 系统要求

| 配置项 | 最低要求 | 推荐配置 |
|--------|---------|---------|
| 操作系统 | Ubuntu 18.04+ | Ubuntu 22.04 LTS |
| 内存 | 16GB | 32GB+ |
| 磁盘空间 | 250GB SSD | 500GB+ SSD |
| 网络 | 稳定外网 | 可访问 Google |

### 1.2 依赖安装

```bash
# Ubuntu/Debian 依赖安装
sudo apt-get update
sudo apt-get install -y \
    git-core gnupg flex bison build-essential zip curl \
    zlib1g-dev libc6-dev-i386 libncurses5 \
    lib32ncurses5-dev x11proto-core-dev libx11-dev \
    lib32z1-dev libgl1-mesa-dev libxml2-utils xsltproc \
    unzip fontconfig

# Python 3
sudo apt-get install -y python3 python3-pip
```

### 1.3 源码下载流程

```bash
# 1. 创建工作目录
mkdir -p ~/aosp && cd ~/aosp

# 2. 安装 Repo 工具
mkdir -p ~/.bin
curl https://storage.googleapis.com/git-repo-downloads/repo > ~/.bin/repo
chmod a+x ~/.bin/repo
export PATH="${HOME}/.bin:${PATH}"

# 3. 初始化仓库（以 Android 13 为例）
repo init -u https://android.googlesource.com/platform/manifest -b android-13.0.0_r1

# 4. 同步源码（耗时数小时）
repo sync -c -j8
```

### 1.4 编译流程

```bash
# 1. 初始化编译环境
cd ~/aosp
source build/envsetup.sh

# 2. 选择编译目标
lunch
# 常用选项：
# aosp_arm64-eng      # ARM64 模拟器，工程版本
# aosp_x86_64-eng     # x86_64 模拟器，工程版本
# aosp_bonito-userdebug  # Pixel 真机版本

# 3. 开始编译（-j 指定并行数）
make -j$(nproc)

# 4. 编译输出位置
# out/target/product/[设备名]/system.img
```

## 二、ADB 调试核心命令

### 2.1 ADB 基础连接

```bash
# 查看连接设备
adb devices -l

# 无线调试连接（需先 USB 配对）
adb tcpip 5555
adb connect 192.168.1.100:5555

# 进入设备 Shell
adb shell

# 指定设备执行命令（多设备时）
adb -s [设备序列号] shell

# 重启设备
adb reboot
adb reboot bootloader    # 进入 fastboot
adb reboot recovery      # 进入 recovery
```

### 2.2 进程与内存查看

```bash
# ===== 进程相关 =====
# 查看所有进程
adb shell ps -A

# 查找特定进程
adb shell ps -A | grep system_server
adb shell ps -A | grep surfaceflinger

# 查看进程线程
adb shell ps -T -p [PID]

# 查看进程详细信息
adb shell cat /proc/[PID]/status
adb shell cat /proc/[PID]/cmdline

# ===== 内存相关 =====
# 查看进程内存映射（C++ 开发者熟悉）
adb shell cat /proc/[PID]/maps

# 查看内存统计
adb shell cat /proc/[PID]/smaps | grep -E "Rss|Pss|Private"

# dumpsys 内存信息
adb shell dumpsys meminfo [PID]
adb shell dumpsys meminfo --package com.android.systemui
```

### 2.3 日志系统操作

```bash
# ===== logcat 基础 =====
# 查看实时日志
adb logcat

# 查看不同缓冲区
adb logcat -b main       # 主日志（应用日志）
adb logcat -b system     # 系统日志
adb logcat -b crash      # 崩溃日志 ⭐重要
adb logcat -b kernel     # 内核日志
adb logcat -b all        # 所有缓冲区

# 按标签过滤
adb logcat -s ActivityManager:D
adb logcat ActivityManager:I *:S  # 只看 ActivityManager Info 及以上

# 按 PID 过滤
adb shell logcat --pid=$(adb shell pidof system_server)

# 清除日志
adb logcat -c

# 保存日志到文件
adb logcat -d > log.txt

# ===== 内核日志 =====
adb shell dmesg          # 查看内核日志
adb shell dmesg -w       # 实时查看内核日志
```

### 2.4 dumpsys 系统服务状态

```bash
# ===== 核心服务状态 =====
# ActivityManager 状态
adb shell dumpsys activity
adb shell dumpsys activity activities    # 查看 Activity 栈
adb shell dumpsys activity processes     # 查看进程信息
adb shell dumpsys activity top           # 查看顶层 Activity

# WindowManager 状态
adb shell dumpsys window
adb shell dumpsys window displays        # 显示信息
adb shell dumpsys window windows         # 窗口列表

# PackageManager 状态
adb shell dumpsys package [包名]

# 查看所有可 dumpsys 的服务
adb shell dumpsys -l

# ===== 其他常用 dumpsys =====
adb shell dumpsys meminfo       # 内存信息
adb shell dumpsys cpuinfo       # CPU 信息
adb shell dumpsys battery       # 电池信息
adb shell dumpsys power         # 电源管理
adb shell dumpsys input         # 输入系统
adb shell dumpsys SurfaceFlinger # 显示合成
```

### 2.5 文件系统操作

```bash
# ===== 文件传输 =====
# 从设备拉取文件
adb pull /data/anr/traces.txt ./
adb pull /data/tombstones/ ./tombstones/

# 推送文件到设备
adb push ./file.txt /data/local/tmp/

# ===== 文件查看 =====
# 查看 init.rc
adb shell cat /system/etc/init/hw/init.rc

# 查看属性
adb shell getprop
adb shell getprop ro.build.version.release
adb shell getprop ro.build.type
adb shell getprop | grep dalvik

# 设置属性（临时，重启失效）
adb shell setprop debug.log.tag.ActivityManager VERBOSE

# ===== 分区信息 =====
adb shell mount | grep system
adb shell df -h
adb shell cat /proc/partitions
```

## 三、init.rc 配置解析

### 3.1 init.rc 位置与结构

```bash
# 查找 init.rc 文件
adb shell find /system/etc/init -name "*.rc"
adb shell ls /system/etc/init/hw/

# 查看主 init.rc
adb shell cat /system/etc/init/hw/init.rc | head -100
```

### 3.2 init.rc 核心语法

```bash
# init.rc 语法结构

# 1. Action（动作）- 在特定触发器时执行
on <trigger>
    <command>
    <command>

# 2. Service（服务）- 定义守护进程
service <name> <pathname> [ <argument> ]*
    <option>
    <option>

# 3. Import（导入）- 导入其他 rc 文件
import <path>
```

### 3.3 关键配置示例解析

```bash
# ===== Action 示例 =====
# 开机启动时执行
on boot
    # 设置基本属性
    setprop ro.build.type userdebug
    
    # 启动核心服务
    start logd
    start servicemanager
    start hwservicemanager
    start vndservicemanager

# 文件系统挂载后执行
on post-fs-data
    # 创建数据目录
    mkdir /data/system 0775 system system
    mkdir /data/misc 01771 system misc

# ===== Service 示例 =====
# Zygote 服务定义
service zygote /system/bin/app_process -Xzygote /system/bin --zygote --start-system-server
    class main                    # 服务类别
    priority -20                  # 优先级（nice值）
    user root                     # 运行用户
    group root readproc reserved_disk  # 运行组
    socket zygote stream 660 root system  # 创建 socket
    onrestart write /sys/android_power/request_state wake
    onrestart restart audioserver
    onrestart restart cameraserver
    writepid /dev/cpuset/foreground/tasks

# surfaceflinger 服务
service surfaceflinger /system/bin/surfaceflinger
    class core
    user system
    group graphics drmrpc readproc
    onrestart restart zygote

# system_server 由 Zygote 启动，不在 init.rc 中定义
```

### 3.4 服务类别（class）说明

| class | 说明 | 控制方式 |
|-------|------|---------|
| `core` | 核心服务，最早启动 | `start core` |
| `main` | 主服务，core 之后启动 | `start main` |
| `late_start` | 延迟启动的服务 | `start late_start` |

```bash
# 查看正在运行的服务
adb shell ls -la /dev/socket/

# 手动启动/停止服务（需要 root）
adb shell stop surfaceflinger
adb shell start surfaceflinger
```

## 四、实操练习任务

### 练习 1：熟悉 ADB 基础命令

```bash
# 任务 1.1：连接设备并查看基本信息
adb devices -l
adb shell getprop ro.product.model
adb shell getprop ro.build.version.release
adb shell getprop ro.build.id

# 任务 1.2：查看关键进程
adb shell ps -A | grep -E "system_server|zygote|surfaceflinger|servicemanager"

# 任务 1.3：查看进程 PID 并获取详细信息
# 1. 获取 system_server PID
adb shell pidof system_server

# 2. 查看该进程的内存映射（前 20 行）
adb shell cat /proc/$(adb shell pidof system_server)/maps | head -20

# 3. 查看进程状态
adb shell cat /proc/$(adb shell pidof system_server)/status
```

### 练习 2：日志分析实践

```bash
# 任务 2.1：查看系统启动日志
adb logcat -d -b system | grep -E "init|Zygote|SystemServer" | head -30

# 任务 2.2：实时监控 Activity 启动
adb logcat -s ActivityManager:I | grep -E "Start proc|Displayed"

# 任务 2.3：查看崩溃日志（如果有）
adb logcat -d -b crash

# 任务 2.4：导出完整日志
adb bugreport ./bugreport.zip
```

### 练习 3：dumpsys 实践

```bash
# 任务 3.1：查看当前 Activity 栈
adb shell dumpsys activity activities | grep -E "Hist #|Task #|topResumedActivity" | head -20

# 任务 3.2：查看所有运行中的进程
adb shell dumpsys activity processes | grep -E "PID|oom" | head -30

# 任务 3.3：查看窗口信息
adb shell dumpsys window windows | grep -E "Window #|mCurrentFocus" | head -20

# 任务 3.4：查看内存信息
adb shell dumpsys meminfo | grep -E "Total|Used|Free" | head -10
```

### 练习 4：init.rc 分析

```bash
# 任务 4.1：导出并查看 init.rc
adb shell cat /system/etc/init/hw/init.rc > ./init.rc

# 任务 4.2：查找所有服务定义
grep "^service" ./init.rc

# 任务 4.3：查找 Zygote 相关配置
grep -A 10 "^service zygote" ./init.rc

# 任务 4.4：查找 class main 的服务
grep -B 1 "class main" ./init.rc | grep "^service"
```

## 五、关键调试命令速查表

### 5.1 进程与线程

| 命令 | 用途 |
|------|------|
| `adb shell ps -A` | 查看所有进程 |
| `adb shell ps -T -p [PID]` | 查看进程线程 |
| `adb shell top` | 实时进程监控 |
| `adb shell pidof [name]` | 根据名称查 PID |
| `adb shell kill -3 [PID]` | 发送 SIGQUIT，生成 traces |

### 5.2 内存分析

| 命令 | 用途 |
|------|------|
| `adb shell dumpsys meminfo` | 系统内存概览 |
| `adb shell dumpsys meminfo [PID]` | 进程内存详情 |
| `adb shell cat /proc/[PID]/maps` | 内存映射 |
| `adb shell cat /proc/[PID]/smaps` | 详细内存统计 |
| `adb shell showmap [PID]` | 图形化内存映射 |

### 5.3 日志相关

| 命令 | 用途 |
|------|------|
| `adb logcat -b crash` | 查看崩溃日志 |
| `adb logcat -b kernel` | 查看内核日志 |
| `adb shell dmesg` | 内核日志 |
| `adb logcat -s [TAG]:D` | 按标签过滤 |
| `adb logcat -t 100` | 查看最近 100 行 |

### 5.4 系统服务

| 命令 | 用途 |
|------|------|
| `adb shell dumpsys activity` | AMS 状态 |
| `adb shell dumpsys window` | WMS 状态 |
| `adb shell dumpsys package` | PMS 状态 |
| `adb shell service list` | 列出所有服务 |
| `adb shell dumpsys -l` | 可 dumpsys 的服务 |
