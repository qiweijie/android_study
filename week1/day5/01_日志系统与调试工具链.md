## 一、Android 日志系统架构

### 1.1 整体架构

```
Android 日志系统架构：

App / Framework / Native                logcat (读取端)
  │                                       ▲
  │ __android_log_write()                 │ 读取日志
  ▼                                       │
liblog (客户端库)                          │
  │                                       │
  │ 通过 socket 写入                       │
  ▼                                       │
┌──────────────────────────────────────────┐
│              logd (守护进程)              │
│  ┌────────┐ ┌────────┐ ┌────────┐      │
│  │ main   │ │ system │ │ crash  │      │
│  │ buffer │ │ buffer │ │ buffer │      │
│  ├────────┤ ├────────┤ ├────────┤      │
│  │ events │ │ kernel │                  │
│  │ buffer │ │ buffer │                  │
│  └────────┘ └────────┘                  │
│                                          │
│  环形缓冲区，固定大小，满了覆盖旧日志     │
└──────────────────────────────────────────┘
```

### 1.2 logd 守护进程

```
logd 的职责：
1. 接收来自 liblog 的日志写入请求
2. 将日志存储在内存中的环形缓冲区
3. 响应 logcat 的读取请求
4. 管理缓冲区大小和日志过期

源码路径：
system/logging/logd/
├── main.cpp          # 入口
├── LogBuffer.cpp     # 缓冲区管理
├── LogReader.cpp     # 日志读取（logcat 连接）
├── LogListener.cpp   # 日志写入监听
└── LogStatistics.cpp # 日志统计
```

## 二、五个日志缓冲区

```
缓冲区      ID    默认大小   用途
──────────────────────────────────────────────────
main        0     256KB     应用层日志（Log.d/i/w/e）
system      1     256KB     系统框架日志（Slog.d/i/w/e）
radio       2     256KB     电话/modem 相关日志
events      3     256KB     结构化事件日志（EventLog）
crash       4     256KB     Crash 堆栈信息
kernel      5     256KB     内核日志（dmesg 内容）
──────────────────────────────────────────────────

稳定性分析时各 buffer 的价值：
• crash  → 最直接：Crash 堆栈、Tombstone 信息
• system → 最常用：AMS/WMS/PMS 等系统服务日志
• main   → 应用行为：应用自身的日志输出
• events → 结构化：Activity 生命周期、ANR 事件
• kernel → 底层：内存不足(OOM)、驱动错误、内核 panic
```

## 三、logcat 高级用法

### 3.1 按缓冲区过滤

```bash
adb logcat -b main        # 只看 main buffer
adb logcat -b system       # 只看 system buffer
adb logcat -b crash        # 只看 crash buffer
adb logcat -b events       # 只看 events buffer
adb logcat -b all          # 所有 buffer
adb logcat -b main,system  # 多个 buffer
```

### 3.2 按 Tag 和级别过滤

```bash
# 格式: tag:level  级别: V D I W E F S
adb logcat ActivityManager:I *:S     # 只看 AMS 的 Info+
adb logcat WindowManager:W *:S      # 只看 WMS 的 Warn+
adb logcat "BinderDemo*:D" *:S      # 通配符

# 多个 Tag
adb logcat ActivityManager:I WindowManager:W *:S
```

### 3.3 按 PID/UID 过滤

```bash
adb logcat --pid=12345             # 按 PID
adb logcat --uid=10086             # 按 UID
```

### 3.4 正则搜索

```bash
adb logcat | grep -E "ANR|crash|FATAL"
adb logcat | grep -i "watchdog"
adb logcat -e "ANR in"             # logcat 内置正则
```

### 3.5 输出格式

```bash
adb logcat -v brief      # 默认：优先级/Tag(PID): 消息
adb logcat -v time       # 带时间戳
adb logcat -v threadtime # 带时间+线程（推荐）
adb logcat -v long       # 详细格式（每条一块）
adb logcat -v color      # 彩色输出

# 推荐的稳定性分析格式
adb logcat -v threadtime -b all
```

### 3.6 日志大小管理

```bash
adb logcat -g                      # 查看各 buffer 大小
adb logcat -G 4M                   # 设置 buffer 大小为 4MB
adb logcat -c                      # 清空所有 buffer
adb logcat -t 100                  # 只显示最近 100 条
adb logcat -T "01-15 10:30:00.000" # 从指定时间开始
```

## 四、Kernel Log

### 4.1 dmesg

```bash
# 查看内核日志
adb shell dmesg
adb shell dmesg | tail -50         # 最近 50 行
adb shell dmesg -T                 # 带可读时间戳
adb shell dmesg -w                 # 实时跟踪（类似 tail -f）

# 过滤关键信息
adb shell dmesg | grep -i "oom"        # OOM killer
adb shell dmesg | grep -i "lowmemory"  # 低内存
adb shell dmesg | grep -i "binder"     # Binder 驱动
adb shell dmesg | grep -i "panic"      # 内核 panic
```

### 4.2 /proc/kmsg

```bash
# 实时读取内核日志（需要 root）
adb shell cat /proc/kmsg
```

### 4.3 Kernel Log 在稳定性中的作用

| 关键字 | 含义 | 稳定性影响 |
|--------|------|-----------|
| `Out of memory` | OOM killer 触发 | 进程被杀 |
| `lowmemorykiller` | LMK 杀进程 | 后台应用被回收 |
| `binder: xxx` | Binder 驱动信息 | Binder 通信异常 |
| `Kernel panic` | 内核崩溃 | 系统重启 |
| `watchdog: BUG` | 内核 watchdog | 内核卡死 |
| `SysRq` | 内核调试按键 | 手动触发 dump |
