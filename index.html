D3
SystemServer & 核心系统服务
深入 Android 的「心脏」—— 稳定性问题的高发区
核心基础 稳定性相关
▼

📖 上午 · 理论学习

  • SystemServer 启动流程：三阶段启动（Bootstrap → Core → Other Services）
  • AMS (ActivityManagerService)：进程管理、四大组件生命周期、OOM Adj 机制
  • PMS (PackageManagerService)：APK 安装/扫描、权限管理
  • WMS (WindowManagerService)：窗口管理、焦点控制 —— ANR 判定的关键角色

🔧 下午 · 动手实操

  • 阅读 SystemServer.java：定位 startBootstrapServices / startOtherServices
  • 用 dumpsys 深入各服务：dumpsys activity、dumpsys window、dumpsys package
  • 追踪 Watchdog 机制：阅读 Watchdog.java 源码，理解 system_server 自检原理
  • 模拟 system_server 卡顿：注入延迟，观察 Watchdog 触发行为
# 核心源码路径
frameworks/base/services/java/com/android/server/SystemServer.java
frameworks/base/services/core/java/com/android/server/am/ActivityManagerService.java
frameworks/base/services/core/java/com/android/server/wm/WindowManagerService.java
frameworks/base/services/core/java/com/android/server/Watchdog.java

  • coredump 分析（如已启用）：用 GDB 加载 coredump 进行离线分析

D8
Watchdog & 系统级稳定性机制
system_server 挂了 = 全系统重启 —— 最严重的稳定性问题
稳定性核心
▼

📖 上午 · 系统级稳定性机制

  • Watchdog 机制：监控线程列表、检查周期 (30s/60s)、触发条件与响应
  • 系统级 Crash 影响链：system_server 崩溃 → Zygote 重启 → 所有 App 重启
  • 低内存管理：LMK / lmkd / OOM Adj 等级，进程优先级与回收策略
  • 进程保活与回收：adj 分级、cached processes、empty processes

🔧 下午 · 深入实操

  • 阅读 Watchdog.java 源码：HandlerChecker 列表、超时判定逻辑
  • 分析 Watchdog 触发的日志：*** WATCHDOG KILLING SYSTEM PROCESS
  • lmkd 机制：/sys/module/lowmemorykiller/parameters/ 配置
  • DropBoxManagerService：系统异常事件持久化存储机制

✅ 当日产出

✓ 理解 Watchdog 完整原理 ✓ 能分析 system_server 重启原因 ✓ 理解 OOM Adj 机制
